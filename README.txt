Revbox - statyczny prototyp frontendu

Zawartość:
- index.html - strona główna
- category.html - listing kategorii (smartfony)
- product.html - karta produktu / recenzja
- assets/css/styles.css - style
- assets/js/app.js - logika interfejsu i mockowane dane
- assets/placeholders/*.jpg - szare placeholdery pod przyszłe grafiki

Placeholdery i docelowe rozmiary:
- hero-wide-1200x320.jpg -> sekcja hero szeroka, rekomendowany eksport 1200 x 320 px
- hero-tall-420x660.jpg -> prawa kolumna hero, rekomendowany eksport 420 x 660 px
- product-card-240x240.jpg -> zdjęcie produktu na kartach, rekomendowany eksport 240 x 240 px
- product-gallery-main-520x420.jpg -> główna grafika produktu, rekomendowany eksport 520 x 420 px
- partner-logo-180x60.jpg -> logo partnera / marketplace, rekomendowany eksport 180 x 60 px
- logo-placeholder-160x48.jpg -> rezerwa pod alternatywne logo, 160 x 48 px
- seo-banner-1200x500.jpg -> rezerwa pod duży baner SEO, 1200 x 500 px
- avatar-64x64.jpg -> rezerwa pod ikonę autora / użytkownika, 64 x 64 px
- category-icon-32x32.jpg -> rezerwa pod ikonę kategorii, 32 x 32 px

Interakcje w mockupie:
- zaznaczanie cech po lewej stronie zmienia match score i sortowanie kart
- modale pomocy i podglądu partnerów działają w JS
- listing i karta produktu korzystają z jednego mockowanego źródła danych w app.js

Kolejny etap:
- podmiana mockowanych tablic JS na API / bazę danych
- panel do zarządzania recenzjami, zaletami, wadami i listą cech per kategoria
